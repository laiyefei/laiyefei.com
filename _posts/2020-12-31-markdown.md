# 语法规则记录

## 段落与换行

~~~
1. 段落的前后必须是空行：
    1.1 空行指的是行内什么都没有，或者只有空白符（空格或制表符）
    1.2 相邻两行文本，如果中间没有空行 会显示在一行中（换行符被转换为空格）
2. 如果需要在段落内加入换行（<br>）：
    2.1 可以在前一行的末尾加入至少两个空格然后换行写其它的文字
3. Markdown 中的多数区块都需要在两个空行之间。
~~~

## 标题

~~~
1. Setext 形式
    H1
    ====
    H2
    ----

    ps: = 和 - 的数量是没有限制的。通常的做法是使其和标题文本的长度相同，这样看起来比较舒服。或者可以像我一样，用四个 - 或 =。Setext 形式只支持 h1 和 h2 两种标题。

2. atx 形式
    2.1 可以用对称的 # 包括文本：
        ####H1####
        #####H2#####
    2.2 也可以只在左边使用 #
        ####H1
        #####H2
    2.3 成对的 # 左侧和只在左边使用的 # 左侧都不可以有任何空白，但其内侧可以使用空白
         ###左侧使用了空格###
        #### 内侧使用了空格

        ps: 在这一点上，可能各种 Markdown 的实现会有不同的结果，不过仍然需要我们遵守语法规则。
~~~

## 引用

~~~
1. 引用内容
    > 引用内容

    ps: 在段落或其他内容前使用 > 符号，就可以将这段内容标记为 '引用' 的内容（<blockquote>）。
2. 多行引用
    >多行引用
    >可以在每行前加 `>` 
    
    ps: 如果引用内容需要换行，
    可以在行尾添加两个空格
    或者在引用内容中加一个空行
3. 嵌套引用
    >也可以在引用中
    >>使用嵌套的引用
4. 其他 Markdown
    >在引用中可以使用使用其他任何 *Markdown* 语法
    
~~~

## 列表

~~~
1. 无序列表
   1.1. * 可以使用 * 作为标记
   1.2. + 也可以使用 +
   1.3. - 或者 -
2. 有序列表
   2.1. 有序列表以数字和 . 开始
   2.2. 数字的序列并不会影响生成的列表序列
   2.3. 但仍然推荐按照自然顺序（1.2.3...）编写
3. 嵌套的列表
   3.1. 第一层 
      3.1.1. + 1-1
      3.1.2. + 1-2
   3.2. 无序列表和有序列表可以随意相互嵌套 
      3.2.1. 2-1
      3.2.2. 2-2
4. 语法和用法
    4.1. 无序列表项的开始是：符号 空格；
    4.2. 有序列表项的开始是：数字 . 空格；
    4.3. 空格至少为一个，多个空格将被解析为一个；
    4.4. 如果仅需要在行前显示数字和 .
    4.5. 可以使用：数字\. 来取消显示为列表
~~~

## 代码

~~~
1. 代码块
    <html> // Tab开头
        <title>Markdown</title>
    </html> // 四个空格开头
    ps. 代码块前后需要有至少一个空行，且每行代码前需要有至少一个 Tab 或四个空格；
2. 行内代码
   也可以通过 ``，插入行内代码（` 是 Tab 键上边、数字 1 键左侧的那个按键）
3. 转换规则
   代码块中的文本（包括 Markdown 语法）都会显示为原始内容，而特殊字符会被转换为 HTML 字符实体。
~~~

## 分隔线

~~~
1. 可以在一行中使用三个或更多的 *、- 或 _ 来添加分隔线（<hr>）
2. 多个字符之间可以有空格（空白符），但不能有其他字符
~~~

## 超链接

~~~
1. 行内式
    [link text](URL 'title text')
2. 参考式
    2.1. 首先，定义链接：[Google][link]
    2.2. 然后定义链接内容：[link]: http://www.google.com/ "Google"
    2.3. 也可以省略 识别符，使用链接文本作为 识别符
    
    ps. 参考式链接的写法相当于行内式拆分成两部分，并通过一个 识别符 来连接两部分。
        参考式能尽量保持文章结构的简单，也方便统一管理 URL。
3. 自动链接
    使用 <> 包括的 URL 或邮箱地址会被自动转换为超链接

    ps. 该方式适合行内较短的链接，会使用 URL 作为链接文字。邮箱地址会自动编码，以逃避抓取机器人。
~~~

## 图片

~~~
1. 图像
    插入图片的语法和插入超链接的语法基本一致，只是在最前面多一个 !。也分为行内式和参考式两种。
    1.1. 行内式
        1.1.1. ![GitHub](https://github.com/laiyefei "laiyefei")
    1.2. 参考式 
        1.2.1. ![GitHub][github]
        1.2.2. [github]: https://github.com
2. 指定图片的显示大小
    Markdown 不支持指定图片的显示大小，不过可以通过直接插入<img />标签来指定相关属性：
~~~

## 强调

~~~
1. 使用 * * 或 _ _ 包括的文本会被转换为 <em></em> ，通常表现为斜体
2. 使用 ** ** 或 __ __ 包括的文本会被转换为 <strong></strong>，通常表现为加粗
3. 用来包括文本的 * 或 _ 内侧不能有空白，否则 * 和 _ 将不会被转换（不同的实现会有不同的表现）
4. 如果需要在文本中显示成对的 * 或 _，可以在符号前加入 \ 即可
5. *、**、_ 和 __ 都必须 成对使用。
~~~

特殊字符包括：

*
_
{}
[]
()
#
+
-
.
!
~~~
